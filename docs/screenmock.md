# 株式材料監視ダッシュボード 画面モック v0.1

## 1. モックの前提

### 1.1 対象

要件定義書 v0.1.3 を前提に、以下の画面モックを作成する。

| ルート                 | 画面名      | 優先度 |
| ------------------- | -------- | --: |
| `/`                 | ホーム画面    |   中 |
| `/disclosures`      | 注目開示画面   |   高 |
| `/watchlist`        | 監視リスト画面  |   高 |
| `/trade-plans`      | 取引計画一覧画面 | 最重要 |
| `/trade-plans/[id]` | 取引計画詳細画面 | 最重要 |
| `/logs`             | 検証ログ画面   |   高 |
| `/settings`         | 設定画面     |   低 |

### 1.2 UI方針

* PCブラウザ優先
* スマホ最適化は後回し
* テーブル中心
* 入力はDialogまたはForm
* 毎日5〜15分で使えることを優先
* 見た目より、判断ミスを防ぐ導線を優先

# 6. 取引計画一覧画面 `/trade-plans`

## 6.1 目的

買い・見送り・監視継続の事前計画を一覧で確認する。

## 6.2 モック

```text
┌──────────────────────────────────────────────────────────────┐
│ 取引計画                                                     │
├──────────────────────────────────────────────────────────────┤
│ [＋取引計画を作成]                                           │
│                                                              │
│ 検索: [銘柄コード・銘柄名・仮説________________]              │
│ 種別: [すべて▼]  状態: [すべて▼]  ロック: [すべて▼]           │
│                                                              │
│ ------------------------------------------------------------ │
│ | 作成日 | コード | 銘柄名 | 種別 | 状態 | エントリー | 損切 | 利確 | 推奨株数 | 予定株数 | ロック | 操作 |
│ | 6/12 | 6659 | メディアリンクス | BUY | DRAFT | 126 | 115 | 150 | 200 | 200 | 未 | [詳細] [確定] |
│ | 6/12 | 9999 | サンプル | NO_TRADE | COMMITTED | - | - | - | - | - | 済 | [詳細] [ログ] |
│                                                              │
│ ------------------------------------------------------------ │
│ 注意表示                                                     │
│ ! DRAFTのBUY計画は、出口条件と推奨株数を確認してから確定してください。 |
└──────────────────────────────────────────────────────────────┘
```

## 6.3 一覧で強調する項目

| 項目   | 理由                                          |
| ---- | ------------------------------------------- |
| 種別   | 買い・見送り・監視継続を区別する                            |
| 状態   | DRAFT / COMMITTED / EXECUTED / CLOSED を把握する |
| 損切価格 | 出口があるかを確認する                                 |
| 利確目標 | 利益側の出口があるかを確認する                             |
| 推奨株数 | リスク逆算の結果を見る                                 |
| 予定株数 | 推奨株数を超えていないか見る                              |
| ロック  | 後から書き換え不可か確認する                              |

* BUY計画では「出口条件」と「リスク計算」を画面中央に置く

* 見送りも正式な計画として扱う

# 10. 主要導線

## 10.1 注目開示から買い計画まで

```text
注目開示を登録
  ↓
材料カテゴリが自動分類される
  ↓
監視リストに追加
  ↓
取引計画を作成
  ↓
BUY計画を入力
  ↓
出口条件を入力
  ↓
リスク逆算サイジングを確認
  ↓
予定株数が推奨株数以内なら確定
  ↓

```

## 10.2 見送り判断の導線


# 11. モック上の重要判断

## 11.1 取引計画画面を最重要画面にする

このアプリの中核は、注目開示一覧ではなく取引計画である。

理由：

* 出口未定の買いを防ぐ
* 損切・利確・最大保有日数を事前固定する
* 許容損失から株数を逆算する
* 確定後に後出し修正できないようにする

---

## 11.2 BUY計画の確定ボタンは条件付きで有効化する

以下を満たすまで、`買い計画を確定する` ボタンは有効化しない。

* 仮説あり
* エントリー予定価格あり
* 損切価格あり
* 利確目標価格あり
* 最大保有日数あり
* 口座資産あり
* 許容リスク%あり
* 推奨株数が1単元以上
* 予定株数が推奨株数以下
* 予定株数が取引単位の倍数

---

## 11.3 COMMITTED後は編集ではなく追記へ誘導する

COMMITTED後は、事前計画の編集フォームを表示しない。

代わりに以下を表示する。

* 確定済みの事前計画
* 実行情報入力
* 検証ログ作成ボタン

---

## 11.4 見送りを軽く扱わない

NO_TRADE計画も `trade_plans` に保存する。

理由：

* 買わない判断を振り返れる
* 過剰取引を抑制できる
* 後から「買わなくてよかった」「買えばよかった」を検証できる
